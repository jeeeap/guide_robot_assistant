[项目仓库：https://github.com/jeeeap/guide\_robot\_assistant](https://github.com/jeeeap/guide_robot_assistant)

实验结果：<https://www.notion.so/36287f61605b801f8eeac6024d1d8ee3?source=copy_link>

## 1. 项目概述

本项目设计并实现一个室内导盲机器人仿真系统，基于 ROS2 Humble、Gazebo Classic 和 TurtleBot3 平台。系统支持自然语言文本指令到自主导航的完整闭环，集成多种路径规划算法和避障策略，并通过 LLM 进行语义理解与任务规划。该项目旨在验证导盲机器人在室内环境中的感知、定位、导航与交互能力，为后续真机部署提供技术支撑。

## 2. 应用场景

### 2.1 场景一：单目标引导导航

场景描述

：
在典型的办公楼/医院/商场等室内环境中，视障用户通过自然语言指令（如"请带我去302房间"或"带我到洗手间"）向机器人发出请求，机器人理解用户意图后，自主规划路径、避开障碍、到达目标位置，并在过程中提供语音反馈。

成功判定标准

：

1. 机器人准确理解用户指令，识别正确的目标位置
2. 机器人从起点安全行驶至目标点，途中无碰撞
3. 到达目标点后给出语音提示（如"已到达302房间"）
4. 任务在合理时间内完成（根据地图大小而定）

量化评估指标

：

- 意图识别准确率
- 导航成功率
- 平均任务耗时
- 路径长度与理论最优路径的比值

### 2.2 场景二：多目标顺序引导

场景描述

：
用户需要依次访问多个地点（如"先去前台登记，然后带我到205诊室，最后到药房取药"）。机器人接收序列指令后，规划最优访问顺序，依次到达每个目标点，并在每个节点提供语音确认。

成功判定标准

：

1. 正确理解多目标序列指令
2. 按合理顺序依次到达所有目标点
3. 每个目标点到达后给出确认反馈
4. 全程无碰撞

量化评估指标

：

- 多目标序列理解准确率
- 按顺序到达率
- 总任务耗时
- 各段路径规划质量

### 2.3 场景三：动态障碍物环境下的避障导航

场景描述

：
在有动态障碍物（如行人、移动的推车等）的走廊中，机器人需要在向目标点行进的同时，灵活避开突然出现的障碍。对比测试两种避障策略的性能。

成功判定标准

：

1. 机器人在动态环境中保持安全行驶
2. 成功避开所有障碍，无碰撞
3. 最终到达目标点
4. 避障过程平稳，无剧烈抖动

量化评估指标

：

- 避障成功率
- 最小安全距离
- 平均避障响应时间
- 任务完成时间增量

## 4. 逻辑架构

系统采用分层架构设计，从下到上依次为：

1. 硬件/仿真层  ：TurtleBot3 机器人、Gazebo 仿真环境、激光雷达、里程计
2. 感知层  ：SLAM 建图、AMCL 定位、环境感知
3. 规划层  ：全局路径规划（A\*、Dijkstra）、局部路径规划（Nav2）
4. 控制层  ：运动控制、避障策略执行
5. 语义层  ：LLM 任务解析、意图理解
6. 交互层  ：文本输入、TTS 反馈

| 层级    | 技术                                    | 作用                       |
| ----- | ------------------------------------- | ------------------------ |

| 建图    | SLAM Toolbox                          | 构建占据栅格地图                 |

## 7. 仿真环境搭建方案

### 7.1 推荐环境

- Ubuntu 22.04
- ROS2 Humble
- Gazebo Classic
- TurtleBot3 Burger
- Navigation2
- SLAM Toolbox
- RViz2
- Python 3

### 7.2 安装依赖

```bash
sudo apt update
sudo apt install -y ros-humble-desktop python3-colcon-common-extensions
sudo apt install -y \
  ros-humble-turtlebot3 \
  ros-humble-turtlebot3-msgs \
  ros-humble-turtlebot3-gazebo \
  ros-humble-navigation2 \
  ros-humble-nav2-bringup \
  ros-humble-slam-toolbox
```

设置模型：

```bash
export TURTLEBOT3_MODEL=burger
```

构建项目：

```bash
cd ~/guide_robot_ws
colcon build --packages-select guide_robot_assistant
source install/setup.bash
```

### 7.3 仿真与建图流程

启动 Gazebo：

```bash
ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py
```

启动 SLAM：

```bash
ros2 launch slam_toolbox online_async_launch.py use_sim_time:=true
```

键盘控制建图：

```bash
ros2 run turtlebot3_teleop teleop_keyboard
```

保存地图：

```bash
ros2 run nav2_map_server map_saver_cli -f ~/guide_robot_ws/src/guide_robot_assistant/maps/guide_map
```

### 7.4 Nav2 与本项目启动

启动 Nav2：

```bash
ros2 launch turtlebot3_navigation2 navigation2.launch.py use_sim_time:=true map:=/home/jp/guide_robot_ws/src/guide_robot_assistant/maps/guide_map.yaml
```

启动文本导航 demo：

```bash
ros2 launch guide_robot_assistant guide_robot_demo.launch.py
```

启动规划对比：

```bash
ros2 launch guide_robot_assistant planner_compare.launch.py
```

启动反应式避障：

```bash
ros2 launch guide_robot_assistant reactive_avoidance.launch.py
```

## 9. 实验结果

### 9.1 路径规划算法对比实验

#### 实验设置

- 地图：Gazebo turtlebot3\_world 生成的占据栅格地图
- 起点与终点：通过 RViz2 的 Publish Point 工具选取
- 对比算法：A\*、Dijkstra

#### 第一组实验结果

| 算法       | 路径长度 | 规划时间 | 扩展节点数 |
| -------- | ---- | ---- | ----- |
| A\*      | 相同   | 更短   | 更少    |
| Dijkstra | 相同   | 较长   | 较多    |

两者路径长度一致，但 A\* 扩展节点更少、规划时间明显更短。

#### 第二组实验结果

| 指标    | A\*     | Dijkstra | 性能提升  |
| ----- | ------- | -------- | ----- |
| 路径长度  | 相同      | 相同       | -     |
| 规划时间  | 0.0145s | 0.1s     | 85.5% |
| 扩展节点数 | 111     | 1000     | 88.9% |

分析

：
A\* 与 Dijkstra 均规划出相同长度路径，说明两者都能找到可行最短路径。但 A\* 由于使用启发式函数引导搜索，扩展节点数仅为 Dijkstra 的约 11.1%，规划时间约为 Dijkstra 的 14.5%，搜索效率明显更高。在动态环境或大地图中，A\* 的优势将更加显著。

### 9.2 自动避障实验

#### 实验设置

- 机器人：TurtleBot3 Burger
- 传感器：2D 激光雷达
- 避障策略：基于激光阈值的三层反应式避障
- 参数：safe\_distance=0.65m, critical\_distance=0.35m

#### 实验结果

| 状态              | min\_front\_distance | avoidance\_active | trigger\_count | 含义                                     |
| --------------- | -------------------- | ----------------- | -------------- | -------------------------------------- |
| forward         | 约 1.0-1.5 m          | false             | 25/26          | 前方距离安全，机器人正常前进                         |
| avoidance\_turn | 约 0.62-0.65 m        | true              | 27             | 前方小于 safe\_distance=0.65m，进入减速转向避障     |
| critical\_turn  | 约 0.12-0.20 m        | true              | 26             | 前方小于 critical\_distance=0.35m，进入紧急原地转向 |

#### 避障策略对比

| 策略        | 优点             | 缺点             | 适用场景       |
| --------- | -------------- | -------------- | ---------- |
| 反应式避障     | 响应快、计算简单、无需地图  | 无全局规划、可能陷入局部极小 | 简单环境、紧急避障  |
| Nav2 局部规划 | 结合全局路径、更平滑、更智能 | 计算量大、依赖地图质量    | 复杂环境、长距离导航 |

分析

：
实验中，当机器人前方障碍距离大于 0.65 m 时，节点保持 forward 状态；当前方距离下降到约 0.64 m 时，节点切换为 avoidance\_turn，开始主动避障；当障碍距离进一步降低到约 0.12-0.20 m 时，节点进入 critical\_turn，停止前进并执行紧急转向。trigger\_count 从 25 增加到 27，说明避障事件被成功检测并记录。

反应式避障适合快速响应紧急情况，而 Nav2 局部规划则更适合在已知地图中进行平滑、智能的导航。在实际系统中，可以结合两者优势，在正常情况下使用 Nav2 规划，在紧急情况下切换到反应式避障。

## 10. 技术要求覆盖方案

### 10.1 感知与环境建模

- 使用 TurtleBot3 2D 激光雷达 `/scan`；
- 使用 SLAM Toolbox 构建占据栅格地图；
- 使用 AMCL 或 SLAM Toolbox 定位；
- 在 RViz2 中显示地图、激光和位姿。

指标

：地图完整度、建图耗时、定位稳定性。

### 10.2 运动控制与避障

- 使用 `/cmd_vel` 实现运动控制；
- 避障策略 1：激光阈值反应式避障；
- 避障策略 2：Nav2 局部规划器避障。

指标

：避障成功率、最小障碍距离、碰撞次数、任务耗时。

### 10.3 路径规划

- 实现 A\*；
- 实现 Dijkstra；
- 使用 Nav2 规划器作为导航基线。

指标

：规划时间、路径长度、扩展节点数、规划成功率。

### 10.4 应用场景

- 场景一：单目标引导导航；
- 场景二：多目标顺序引导；
- 场景三：障碍物绕行与避障对比。

每个场景均有成功标准和量化指标。

### 10.5 人机交互

- 主接口为文本自然语言输入；
- 输出为 TTS 文本反馈；
- 后续可扩展 ASR 语音输入。

指标

：解析准确率、响应时间、未知命令处理能力。

### 10.6 LLM / 基础模型

- `center_node` 支持 LLM 任务解析；
- 输出结构化 JSON；
- 失败时规则解析兜底；
- 记录 LLM 延迟、成功状态和错误类型。

指标

：LLM 推理延迟、JSON 合法率、意图识别准确率、失败案例数量。

### 10.7 系统与部署分析

TurtleBot3 适合室内低速导盲导航仿真，成本较低、资料完善。真实部署时的主要问题包括传感器噪声、轮式底盘打滑、执行器延迟、麦克风噪声、网络 API 延迟和真实行人动态行为复杂。应对策略包括限制速度、设置安全距离、加入本地规则兜底、使用 rosbag 回放验证以及在真实部署前进行多轮安全测试。

## 11. 技术路线

### 阶段 1：仿真平台验证

启动 TurtleBot3 Gazebo，验证 `/scan`、`/odom`、`/cmd_vel` 和键盘控制。

### 阶段 2：SLAM 建图与定位导航

使用 SLAM Toolbox 建图，保存 `guide_map.yaml` 和 `guide_map.pgm`，启动 Nav2 并在 RViz2 中验证导航。

### 阶段 3：文本指令导航闭环

启动 `guide_robot_demo.launch.py`，输入"去302"，验证文本命令、语义解析、Nav2 导航、TTS 反馈和 CSV 记录。

### 阶段 4：路径规划对比

启动 `planner_compare.launch.py`，发布起点终点，在 RViz2 中显示 A\* 和 Dijkstra 路径，记录规划时间、路径长度和扩展节点数。

### 阶段 5：避障策略对比

分别测试 `reactive_avoidance_node` 和 Nav2 局部规划避障，记录避障触发次数、最小障碍距离、碰撞次数和任务耗时。

### 阶段 6：LLM 解析实验

配置 API Key，启用 LLM 解析，输入多种自然语言命令，记录 LLM 输出 JSON、延迟、成功率和失败案例。

### 阶段 7：综合演示与报告整理

完成场景一、场景二、场景三演示，整理系统架构图、节点通信图、实验表格、失败案例和部署分析。

## 12. 风险与解决方案

| 风险                   | 解决方案                                 |
| -------------------- | ------------------------------------ |
| Gazebo/RViz2 图形性能不稳定 | 使用简单世界、降低场景复杂度，必要时提前录制演示             |
| Nav2 定位或导航失败         | 改善建图质量，准确设置初始位姿，先手动测试目标点             |
| LLM API 延迟或失败        | 保留规则解析兜底，记录错误类型                      |
| `/cmd_vel` 控制冲突      | 反应式避障和 Nav2 不同时运行                    |
| 目标点坐标不准              | 使用 RViz2 标定后更新 `task_locations.yaml` |
