# 阶段二：系统方案与架构设计

## 1. 项目概述

### 1.1 项目名称

**基于 ROS2 + Gazebo 的室内导盲机器人智能导航仿真系统**

### 1.2 项目背景与目标

本项目面向《机器人技术》课程期末项目，选择“室内导盲机器人”作为移动机器人应用场景。系统基于 ROS2 Humble、Gazebo、TurtleBot3、Navigation2、SLAM Toolbox 和大语言模型构建，目标是在仿真室内环境中实现从用户自然语言指令到机器人自主导航执行的完整闭环。

机器人需要完成环境感知、建图定位、路径规划、避障导航、人机交互、LLM 语义解析和实验数据记录。项目不依赖真实硬件，通过 Gazebo 完成物理仿真，通过 RViz2 展示地图、激光雷达、位姿、路径和目标点。

### 1.3 当前工程基础

ROS2 工作空间：

```text
~/guide_robot_ws
```

核心包：

```text
guide_robot_assistant
```

已实现节点与文件：

| 文件 | 功能 |
|---|---|
| `text_command_node.py` | 文本自然语言输入，发布 `/raw_text` |
| `center_node.py` | LLM/规则语义解析，发布导航命令和反馈文本 |
| `nav_client_node.py` | 调用 Nav2 `NavigateToPose` action 执行导航 |
| `tts_node.py` | 订阅 `/tts_text`，输出机器人反馈文本 |
| `experiment_logger_node.py` | 记录任务耗时、成功失败、LLM 延迟和错误类型 |
| `planner_compare_node.py` | 在 `/map` 上实现 A* 与 Dijkstra 规划对比 |
| `reactive_avoidance_node.py` | 基于 `/scan` 的激光阈值反应式避障 |
| `config/task_locations.yaml` | 保存目标点坐标 |
| `launch/guide_robot_demo.launch.py` | 启动文本导航主链路 |
| `launch/planner_compare.launch.py` | 启动规划对比节点 |
| `launch/reactive_avoidance.launch.py` | 启动反应式避障节点 |

## 2. 具体应用场景

### 2.1 场景一：单目标导盲引导导航

用户在室内入口处，希望机器人带其前往某个房间。用户输入：

```text
请带我去302房间
```

系统流程：

1. `text_command_node` 发布 `/raw_text`；
2. `center_node` 将自然语言解析为结构化任务；
3. 输出示例：

```json
{"intent":"navigate","targets":["room_302"],"reply":"好的，我将带您前往302房间。"}
```

4. `nav_client_node` 查询 `task_locations.yaml` 中 `room_302` 的坐标；
5. 调用 Nav2 `NavigateToPose`；
6. TurtleBot3 在 Gazebo 中导航到目标点；
7. `experiment_logger_node` 记录实验结果。

成功标准：机器人到达目标点 0.5 m 范围内，且无碰撞。

评价指标：任务耗时、导航成功率、路径长度、LLM 解析延迟、指令解析准确率、失败原因。

### 2.2 场景二：多目标顺序引导

用户希望机器人依次前往多个地点，例如：

```text
依次去302和327
```

系统解析为：

```json
{"intent":"multi_navigate","targets":["room_302","room_327"],"reply":"好的，我将依次带您前往302房间、327房间。"}
```

`nav_client_node` 将目标加入队列，先导航到 `room_302`，成功后自动导航到 `room_327`。

成功标准：机器人按顺序到达所有目标点。

评价指标：总耗时、目标完成率、导航失败次数、避障触发次数、重规划次数。

### 2.3 场景三：障碍物绕行与避障策略对比

在机器人路径中放置静态或动态障碍物，对比两种避障策略：

- 策略 A：`reactive_avoidance_node` 基于激光雷达距离阈值避障；
- 策略 B：Nav2 局部规划器避障。

成功标准：机器人不碰撞并到达目标点。

评价指标：避障成功率、最小障碍距离、碰撞次数、任务完成时间、路径平滑性。

## 3. 系统基本功能

1. **仿真运行**：在 Gazebo 中加载 TurtleBot3，并通过 ROS2 控制机器人运动。
2. **环境感知**：使用 TurtleBot3 2D 激光雷达 `/scan` 感知障碍物。
3. **建图**：使用 SLAM Toolbox 构建占据栅格地图。
4. **定位**：使用 AMCL 或 SLAM Toolbox 估计机器人在地图中的位姿。
5. **自然语言交互**：用户通过文本输入导航命令，后续可扩展语音输入。
6. **LLM 语义解析**：`center_node` 调用 LLM 输出结构化 JSON，失败时规则解析兜底。
7. **目标点导航**：根据 `task_locations.yaml` 中坐标调用 Nav2 导航。
8. **多目标任务**：支持按顺序执行多个目标点导航。
9. **路径规划对比**：实现 A* 和 Dijkstra，并发布路径到 RViz2。
10. **避障策略对比**：实现反应式避障，并与 Nav2 局部规划避障对比。
11. **反馈输出**：通过 `/tts_text` 和 `tts_node` 输出机器人反馈。
12. **实验记录**：记录任务耗时、成功失败、LLM 延迟、错误类型等数据。

## 4. 逻辑架构

### 4.1 主导航逻辑

```text
用户自然语言指令
        ↓
text_command_node
        ↓ /raw_text
center_node
        ↓ LLM解析 / 规则兜底
/navigation_command
        ↓
nav_client_node
        ↓ NavigateToPose Action
Navigation2
        ↓ /cmd_vel
Gazebo TurtleBot3
        ↓
/navigation_status + /tts_text
        ↓
experiment_logger_node + tts_node
```

### 4.2 路径规划对比逻辑

```text
/map + 起点终点
        ↓
planner_compare_node
        ↓
A* 搜索 / Dijkstra 搜索
        ↓
/astar_path /dijkstra_path
        ↓
RViz2 可视化
        ↓
/planner_compare_result
```

### 4.3 避障对比逻辑

反应式避障：

```text
/scan → reactive_avoidance_node → 距离阈值判断 → /cmd_vel → /avoidance_status
```

Nav2 避障：

```text
/map + /scan + 目标点 → Nav2局部代价地图 + 局部规划器 → /cmd_vel
```

两类避障策略分别运行，避免多个节点同时竞争 `/cmd_vel`。

## 5. ROS2 节点与接口

| 节点 | 输入 | 输出 | 作用 |
|---|---|---|---|
| `text_command_node` | 终端文本 | `/raw_text` | 输入自然语言命令 |
| `center_node` | `/raw_text` | `/navigation_command`, `/tts_text`, `/task_status` | 语义解析与任务调度 |
| `nav_client_node` | `/navigation_command` | `/navigation_status`, Nav2 action | 发送导航目标 |
| `tts_node` | `/tts_text` | `/tts_status` | 输出反馈文本 |
| `experiment_logger_node` | `/raw_text`, `/task_status`, `/navigation_status` | CSV | 记录实验数据 |
| `planner_compare_node` | `/map`, `/planner_compare_request` | `/astar_path`, `/dijkstra_path`, `/planner_compare_result` | 规划对比 |
| `reactive_avoidance_node` | `/scan` | `/cmd_vel`, `/avoidance_status` | 反应式避障 |

主要接口：

| 名称 | 类型 | 说明 |
|---|---|---|
| `/scan` | `sensor_msgs/LaserScan` | 激光雷达数据 |
| `/map` | `nav_msgs/OccupancyGrid` | 占据栅格地图 |
| `/cmd_vel` | `geometry_msgs/Twist` | 速度控制 |
| `/raw_text` | `std_msgs/String` | 用户文本命令 |
| `/navigation_command` | `std_msgs/String` | JSON 导航命令 |
| `/navigate_to_pose` | `nav2_msgs/NavigateToPose` | Nav2 action |
| `/astar_path` | `nav_msgs/Path` | A* 路径 |
| `/dijkstra_path` | `nav_msgs/Path` | Dijkstra 路径 |
| `/avoidance_status` | `std_msgs/String` | 避障状态 |

## 6. 技术架构

| 层级 | 技术 | 作用 |
|---|---|---|
| 操作系统 | Ubuntu 22.04 | ROS2 运行环境 |
| 中间件 | ROS2 Humble | 节点通信、Topic、Action、Launch |
| 仿真 | Gazebo Classic | TurtleBot3 物理仿真 |
| 机器人平台 | TurtleBot3 Burger | 差速移动机器人模型 |
| 可视化 | RViz2 | 地图、路径、激光和位姿显示 |
| 建图 | SLAM Toolbox | 构建占据栅格地图 |
| 定位 | AMCL / SLAM Toolbox | 机器人位姿估计 |
| 导航 | Navigation2 | 全局规划、局部规划、控制与恢复 |
| 自定义算法 | A*, Dijkstra, 反应式避障 | 规划和避障对比 |
| 基础模型 | 智谱 GLM 或兼容 Chat Completions API 的 LLM | 自然语言任务解析 |
| 开发语言 | Python 3 + rclpy | 自定义 ROS2 节点开发 |

LLM 位于语义规划层，只负责将自然语言转换为结构化任务，不参与底层实时运动控制，以降低延迟和安全风险。

## 7. 仿真环境搭建方案

### 7.1 推荐环境

- Ubuntu 22.04
- ROS2 Humble
- Gazebo Classic
- TurtleBot3 Burger
- Navigation2
- SLAM Toolbox
- RViz2
- Python 3

### 7.2 安装依赖

```bash
sudo apt update
sudo apt install -y ros-humble-desktop python3-colcon-common-extensions
sudo apt install -y \
  ros-humble-turtlebot3 \
  ros-humble-turtlebot3-msgs \
  ros-humble-turtlebot3-gazebo \
  ros-humble-navigation2 \
  ros-humble-nav2-bringup \
  ros-humble-slam-toolbox
```

设置模型：

```bash
export TURTLEBOT3_MODEL=burger
```

构建项目：

```bash
cd ~/guide_robot_ws
colcon build --packages-select guide_robot_assistant
source install/setup.bash
```

启动 Gazebo：

```bash
ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py
```

启动 SLAM：

## 8. 数据集与模型资源

### 8.1 地图数据

地图由 Gazebo 仿真环境中机器人通过 SLAM 自主生成，不依赖外部公开数据集。输出文件：

```text
maps/guide_map.yaml
maps/guide_map.pgm
```

### 8.2 目标点数据

目标点配置文件：

```text
config/task_locations.yaml
```

保存每个地点的名称、x、y 和 yaw。实测阶段需要通过 RViz2 标定真实坐标。

### 8.3 传感器与状态数据

| 数据 | 用途 |
|---|---|
| `/scan` | 建图、避障、局部代价地图 |
| `/odom` | 里程计与运动估计 |
| `/map` | 路径规划、定位、导航 |
| `/amcl_pose` | 定位结果 |
| `/cmd_vel` | 控制机器人运动 |

### 8.4 模型资源

使用智谱 GLM 或兼容 Chat Completions API 的 LLM 作为基础模型，用于自然语言任务解析。API Key 通过环境变量配置：

```bash
export ZHIPUAI_API_KEY="your_api_key"
```

若未配置 API Key，系统自动使用规则解析兜底。

### 8.5 实验数据

默认 CSV 日志路径：

```text
~/guide_robot_ws/results/experiment_log.csv
```

字段包括：

```text
experiment_id, scenario, raw_text, intent, targets, start_time, end_time,
time_cost, success, final_state, error_type, action_status,
parser_source, llm_latency, llm_success, llm_error
```

规划结果通过 `/planner_compare_result` 输出，避障结果通过 `/avoidance_status` 输出。

## 9. 技术要求覆盖方案

### 9.1 感知与环境建模

- 使用 TurtleBot3 2D 激光雷达 `/scan`；
- 使用 SLAM Toolbox 构建占据栅格地图；
- 使用 AMCL 或 SLAM Toolbox 定位；
- 在 RViz2 中显示地图、激光和位姿。

指标：地图完整度、建图耗时、定位稳定性。

### 9.2 运动控制与避障

- 使用 `/cmd_vel` 实现运动控制；
- 避障策略 1：激光阈值反应式避障；
- 避障策略 2：Nav2 局部规划器避障。

指标：避障成功率、最小障碍距离、碰撞次数、任务耗时。

### 9.3 路径规划

- 实现 A*；
- 实现 Dijkstra；
- 使用 Nav2 规划器作为导航基线。

指标：规划时间、路径长度、扩展节点数、规划成功率。

### 9.4 应用场景

- 场景一：单目标引导导航；
- 场景二：多目标顺序引导；
- 场景三：障碍物绕行与避障对比。

每个场景均有成功标准和量化指标。

### 9.5 人机交互

- 主接口为文本自然语言输入；
- 输出为 TTS 文本反馈；
- 后续可扩展 ASR 语音输入。

指标：解析准确率、响应时间、未知命令处理能力。

### 9.6 LLM / 基础模型

- `center_node` 支持 LLM 任务解析；
- 输出结构化 JSON；
- 失败时规则解析兜底；
- 记录 LLM 延迟、成功状态和错误类型。

指标：LLM 推理延迟、JSON 合法率、意图识别准确率、失败案例数量。

### 9.7 系统与部署分析

TurtleBot3 适合室内低速导盲导航仿真，成本较低、资料完善。真实部署时的主要问题包括传感器噪声、轮式底盘打滑、执行器延迟、麦克风噪声、网络 API 延迟和真实行人动态行为复杂。应对策略包括限制速度、设置安全距离、加入本地规则兜底、使用 rosbag 回放验证以及在真实部署前进行多轮安全测试。

## 10. 技术路线

### 阶段 1：仿真平台验证

启动 TurtleBot3 Gazebo，验证 `/scan`、`/odom`、`/cmd_vel` 和键盘控制。

### 阶段 2：SLAM 建图与定位导航

使用 SLAM Toolbox 建图，保存 `guide_map.yaml` 和 `guide_map.pgm`，启动 Nav2 并在 RViz2 中验证导航。

### 阶段 3：文本指令导航闭环

启动 `guide_robot_demo.launch.py`，输入“去302”，验证文本命令、语义解析、Nav2 导航、TTS 反馈和 CSV 记录。

### 阶段 4：路径规划对比

启动 `planner_compare.launch.py`，发布起点终点，在 RViz2 中显示 A* 和 Dijkstra 路径，记录规划时间、路径长度和扩展节点数。

### 阶段 5：避障策略对比

分别测试 `reactive_avoidance_node` 和 Nav2 局部规划避障，记录避障触发次数、最小障碍距离、碰撞次数和任务耗时。

### 阶段 6：LLM 解析实验

配置 API Key，启用 LLM 解析，输入多种自然语言命令，记录 LLM 输出 JSON、延迟、成功率和失败案例。

### 阶段 7：综合演示与报告整理

完成场景一、场景二、场景三演示，整理系统架构图、节点通信图、实验表格、失败案例和部署分析。

## 11. 风险与解决方案

| 风险 | 解决方案 |
|---|---|
| Gazebo/RViz2 图形性能不稳定 | 使用简单世界、降低场景复杂度，必要时提前录制演示 |
| Nav2 定位或导航失败 | 改善建图质量，准确设置初始位姿，先手动测试目标点 |
| LLM API 延迟或失败 | 保留规则解析兜底，记录错误类型 |
| `/cmd_vel` 控制冲突 | 反应式避障和 Nav2 不同时运行 |
| 目标点坐标不准 | 使用 RViz2 标定后更新 `task_locations.yaml` |
